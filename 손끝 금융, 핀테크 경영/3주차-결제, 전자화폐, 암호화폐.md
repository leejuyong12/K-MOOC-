## 결제, 전자화폐, 암호화폐



### 3-2. 온라인 결제시장의 동향

- 온라인 결제의 유형

  - ![캡처](md-images/%EC%BA%A1%EC%B2%98-1634977717010.PNG)
  - 고객 - 고객 > 개인간 송금
  - 고객 - 비즈니스(기업) > 구매대금 결제
  - 비즈니스(기업) - 고객 > 상인 지급
  - 비즈니스(기업) - 비즈니스(기업) > 기업간 거래

- 결제 서비스와 공급업자(McKinsey, SWIFT, 2018.10)

  - 고객과 기업 간의 전자상거래, 온라인 마켓플레이스, 고지서 결제
  - 기업과 고객 간의 마켓플레이스 지불
  - 기업과 기업 간의 중소기업 거래 분야
  - 현재 성장이 가속되고 있는 결제서비스 분야

- 결제 시장을 둘러싼 경쟁

  - 블록체인, P2P(Peer-to-Peer) 네트워크
    - 금융기관을 완전히 우회하고, 당사자 간의 직접 결제를 허용하기 때문에 기존 시스템을 붕괴시킬 잠재력을 가짐
  - 오픈뱅킹 및 마이데이터
    - 고객 데이터와 관리에 대한 은행의 독점적 관행이 약화됨
    - 기존의 은행시스템을 이용하지 않는 새로운 형태의 핀테크 결제서비스 공급업자들이 등장하기 시작함.

- 온라인 결제시장의 동향

  - B2B결제 시장
    - B2B 송금
      - 대부분의 B2B 거래는 자국 내에서 진행
      - 대부분의 B2B 비즈니스는 종이 기반 처리
    - 국경 간 B2B 송금
      - 글로벌화로 인해 기업의 해외 비즈니스 장애물이 제거되는 중
    - 외국환은 여전한 도전 과제
    - 국경 간 결제의 변화
      - 분산원장 기술, 카드 및 네트워크 혁신 등 기술 압박
      - 규제 및 제재 프레임워크 변화
      - 소매 및 기업 대상 글로벌 상거래 가속화
      - 고객의 요구 변화
    - 기술 기반 B2B 결제 시장은 디지털 중심의 민첩하고 새로운 핀테크 사업자들에게 큰 기회가 될 것
      - 국경 간 송금시 문제사항
        - 송금수수료
        - 적용환율
        - 보이지 않는 부대 비용
        - 송금 속도
      - 핀테크 기업들
        - 달러, 유로 같은 보편적 통화를 전통적 은행에 비해 더 저렴한 송금 서비스를 제공하여 경쟁력을 키움

- 소매업자 결제 시장

  - 결제 전략

    - 고객의 충성도를 높이고 매출이나 이익을 높이는 방법이 될 수 있음

    - 매끄러운 결제 전략을 내세우고 있음

  - 매끄럽지 못한 결제 경험

    - 선호하는 결제 양식이 제공되지 않음
    - 결제가 거부됨
    - 민감한 정보를 요구함
    - 회원이 아닌 게스트의 결제는 지원하지 않음
    - 결제를 위해 다른 사이트로 이동하도록 함
    - 별도의 수수료를 요구함

  - ![캡처](md-images/%EC%BA%A1%EC%B2%98-1634978328878.PNG)

  - 소매업자 결제 전략 수립 고려 사항

    - 결제 서비스에 대한 고객의 요구는 무엇인가?
    - 부가가치 서비스 개념으로 결제를 어떻게 활용할 것인가?
    - 어떤 결제 방법을 선택할 것인가?
    - 어떤 방식의 금융 지원을 제공할 것인가?
    - 공급업체는 어떤 기준으로 정할 것인가?
    - 보안과 관리의 수준은 어디에 맞출 것인가?
    - 데이터 보호라는 글로벌 트렌드를 어떻게 반영할 것인가?

- 인터넷 신용카드 지급결제 시스템

  - Green Commerce Model(First Virtual, 1995)
  - SET(Secure Electronic Transaction : Visa, Mastercard, 1996)

  - 카드회원 본인 인증
  - 상인 또는 PG의 고객 신용카드 정보 획득 방지
  - 거래 부인 방지

- SSL과 SET

  - SSL 기반 시스템(TLS라고도 함)
  - ![캡처](md-images/%EC%BA%A1%EC%B2%98-1634989649175.PNG)
  - SSL 기반 시스템
    - 지불정보
      - 기밀성, 인증, 부인방지가 이루어져야 함
    - 지불 승인
      - 인증, 무결성 등의 보안이 이루어져야 함
      - 암호화를 통해 해결
    - 장점
      - 단순, 명료
      - 개발 및 유지보수 용이
      - 고객 사용 편리
    - 단점
      - 카드번호, 유효기간, CVC번호, 비밀번호 등 카드정보가 상인에게 노출
      - 카드회원 인증의 문제
      - 보안성 낮음
  - Secure Electronic Transaction(SET)
    - 전자상거래에서 지불정보를 안전하고 비용 효과적으로 처리할 수 있도록 규정한 프로토콜
    - 비자와 마스터카드가 주도하여 규정한 신용카드 지급 결제 프로토콜
    - 지금은 사라짐
    - 장점
      - 지불정보가 상인에게 노출되지 않도록 이중(dual)으로 암호화
      - 보안성 높음(Key size 큼, 인증, 무결성..)
    - 단점
      - 시스템이 복잡해지고 시스템 부하가 큼
      - 별도의 고객 소프트웨어(Digital Wallet) 필요함
      - 사용 불편
      - 유지보수 어려움

  - 3-D SET
    - SET에서의 고객 소프트웨어(Digital Wallet) 기능을 웹 서버에 올림
    - 지금은 사라짐
      - 장점
        - 고객 편리성 증대
      - 단점
        - 상인, 카드사(PG), 인증 기관 시스템의 복잡성은 계속 남음
        - 고객의 웹 브라우저와 고객 서버 간의 보안성 문제

- 3-D Secure와 SPA

  - SET 실패 이후에 비자와 마스터카드는 독자적으로 신용카드 지급결제 시스템을 개발한다.
  - 3-D Secure(Visa)

- 3-D Secure vs SPA
  - 3-D Secure
    - 중앙집중식 구조

### 3-4. 스마트폰 간편결제

- 간편결제
  - 모바일 기기에 저장된 생체정보나 신용카드 정보 등을 이용해 온오프라인 상거래에서 빠르고 간편하게 결제할 수 있는 전자 결제 서비스
- 신용카드 기반 간편결제
  - 일반 신용카드- 기존 플라스틱 카드
  - 모바일카드(USIM 카드) - 휴대전화의 USIM에 저장하여 결제에 사용할 수 있는 신용카드와 체크카드

### 3-5. 전자화폐, 암호화폐

- 전자화폐
  - 디지털화폐(digital money, digital currency)
    - 디지털 방식으로만 사용할 수 있는 유형의 화폐
  - 전자화폐(electronic money, electronic currency)
    - 디지털 화폐 중에서 범용성의 요건을 갖춘 유형의 화폐
  - 가상화폐(virtual currency)
    - 정부의 통제를 받지 않으면서 개발자가 발행, 관리하고 특정한 가상 커뮤니티에서만 통용되는 결제 수단
  - 암호화폐(cryptocurrency)
    - 블록체인 기술로 암호화되어 분산발행되고, 일정한 네트워크에서 화폐로 사용할 수 있는 전자정보
- 전자화폐의 특징 및 문제점
  - 특징
    - 화폐 가치를 저장, 전송 가능한 디지털 형태로 표현
    - 실제 화폐 사용 방법을 전자적으로 구현
    - 실제 화폐 사용에 비해 적은 처리 비용
  - 문제점
    - 개인 정보 보호 문제(Privacy, 익명성)
    - 화폐의 중복 사용 문제(불법적 복사)
- 전자화폐 역사
  - 1994 David Chaum이 설립(네덜란드)
  - 'Ecash(단위:cyberbuck)' 라고 하는 전자 화폐 발행
  - 전자화폐의 문제점 해결
    - Privacy 보호에 역점 : Blind Signature
    - 화폐 불법 복사, 중복 사용 문제 : 화폐 일련번호 DB로 해결
- IC 카드형 전자 화폐
  - IC카드에 화폐가치를 저장하여 지급 수단으로 사용
  - IC카드의 유형
    - 비접촉식 카드(ex 충전식 교통카드)
    - 하이브리드 카드
    - 콤비 카드
    - 통합 카드

- 암호화폐
  - 분산 원장(Distributed Ledger)에서 공개키 암호화를 통해 안전하게 전송하고, 해시 함수를 이용해 쉽게 소유권을 증명해낼 수 있는 디지털 자산
  - ex) 비트코인
- 분산 원장(공유 원장)
  - 지리적으로 여러 사이트나, 여러 국가 또는 여러 기관에 분산되어 복제, 공유, 동기화 되는 데이터베이스
  - 중앙집중적인 관리자나 데이터 저장소가 존재하지 않음
  - 사용자 개인 간 직접 접속(peer-to-peer) 네트워크 필요
  - 노드 간 복제 데이터에 대한 합의 알고리즘 필요
- 블록체인
  - 분산원장 관리 기술의 한 종류
  - '블록'이라고 부르는 소규모 데이터들을 체인 형태로 연결하여 수많은 컴퓨터를 동시에 복제한 다음 저장하는 기술
  - 분산 데이터 저장 환경을 사용하기 때문에, 누구도 임의로 수정할 수 없고, 누구나 변경 결과를 열람할 수 있음
  - 모든 거래 내역이 분산, 복제, 공유, 동기화 되면서 위조가 불가능함
- 비트코인의 특징
  - 비트코인 거래시 디지털 서명을 사용하기 때문에 거래 내역을 알 수 있음
- 익명성에 대한 요구 증가
  - 좀 더 발전된 형태의 익명성을 제공하는 암호화폐들도 등장
  - Dash, Monero, Zcash
- 블록체인 응용 사례
  - Decentralized identifiers(DIDs)
  - 스마트폰 등 개인기기에 신원정보를 분산해 관리하는 디지털 신분증 시스템
  - 블록체인의 특성 상 각자의 정보를 제 3의 기관이나 기업에 저장하지 않아도 됨
  - 모바일 신분증이라는 이름으로 시장 확대 및 경쟁 격화
